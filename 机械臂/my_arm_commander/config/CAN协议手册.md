# 42Motor CAN 通信协议手册

## 命令速查表

### 运行时命令（0x00~0x0F）

| 命令码 | 名称 | 说明 |
|--------|------|------|
| `0x01` | 使能电机 | `1` = 使能 / `0` = 停止 |
| `0x02` | 编码器校准 | 触发自动校准流程 |
| `0x03` | 设置电流 | 切换到电流闭环模式 |
| `0x04` | 设置速度 | 切换到速度闭环模式 |
| `0x05` | 设置位置 | 切换到位置闭环模式 |
| `0x06` | 设置位置（带时间） | 指定运动时长 |
| `0x07` | 设置位置（带限速） | 指定最大速度限制 |

### 配置命令（0x10~0x1F）

| 命令码 | 名称 | 说明 |
|--------|------|------|
| `0x11` | 设置节点 ID | 修改 CAN 节点地址 |
| `0x12` | 设置电流限制 | 最大电流保护 |
| `0x13` | 设置速度限制 | 最大速度保护 |
| `0x14` | 设置加速度 | 加减速斜率 |
| `0x15` | ⭐ **设置零位（归位）** | 当前位置记为零点，自动存 EEPROM |
| `0x16` | 上电自动使能 | 上电后自动进入运行模式 |
| `0x17` | 设置 Kp | 位置环比例增益 |
| `0x18` | 设置 Kv | 位置环速度前馈 |
| `0x19` | 设置 Ki | 位置环积分增益 |
| `0x1A` | 设置 Kd | 位置环微分增益 |
| `0x1B` | 堵转保护开关 | 开启/关闭堵转检测 |

### 查询命令（0x20~0x2F）

| 命令码 | 名称 | 返回数据 |
|--------|------|----------|
| `0x21` | 查询电流 | 当前电流 (A) + 完成状态 |
| `0x22` | 查询速度 | 当前速度 (r/s) + 完成状态 |
| `0x23` | 查询位置 | 当前位置 (转) + 完成状态 |
| `0x24` | 查询零位偏移 | 编码器偏移量 (ticks) |
| `0x25` | 查询温度 | 芯片温度 (℃) |

### 系统命令

| 命令码 | 名称 | 说明 |
|--------|------|------|
| `0x7D` | 开启温度监测 | 1Hz 周期采集温度 |
| `0x7E` | 恢复出厂设置 | 擦除 EEPROM 并重启 |
| `0x7F` | 软件复位 | 立即重启 |

> **CAN ID 计算公式**：`CAN_ID = (NodeID << 7) | 命令码`，节点 ID 默认 `0`（广播）。
> 下方示例均以节点 ID = `0` 为例，发给其他节点时请自行换算 CAN ID。

## 概述

本固件使用 **标准 CAN 帧（11-bit ID）** 进行通信，数据长度固定为 **8 字节（DLC=8）**。

### CAN ID 编码规则

```
CAN ID (11-bit) = (NodeID << 7) | Command
```

- **高 4 位（bit10~bit7）**：节点 ID，范围 0~15
- **低 7 位（bit6~bit0）**：命令码
- 节点 ID 为 `0` 时为广播模式，所有电机都会响应
- 默认节点 ID 由硬件拨码开关（PA8/PA9/PA10）决定，上电时读取

### 数据编码规则

数据采用 **小端字节序（Little-Endian）**：

| 类型 | 字节 | 说明 |
|------|------|------|
| `float` | 4 字节 | IEEE 754 单精度浮点数 |
| `int32_t` | 4 字节 | 有符号 32 位整数 |
| `uint32_t` | 4 字节 | 无符号 32 位整数 |

## SocketCAN 使用方法

### 1. 配置 CAN 接口

```bash
# 设置波特率并启用 CAN 接口
sudo ip link set can0 type can bitrate 1000000
sudo ip link set up can0

# 查看接口状态
ip -details link show can0
```

### 2. 发送命令格式

```bash
cansend can0 <CAN_ID>#<DATA>
```

- `CAN_ID`：3 位十六进制数（标准帧 11 位）
- `DATA`：16 位十六进制数（8 字节）

### 3. 监听返回数据

```bash
candump can0
```

### 4. 浮点数与十六进制互转（Python 辅助）

```python
import struct

# 浮点数 → 十六进制字符串（小端）
def float_to_hex(f):
    return struct.pack('<f', f).hex()

# 十六进制字符串 → 浮点数（小端）
def hex_to_float(h):
    return struct.unpack('<f', bytes.fromhex(h))[0]

# 示例
print(float_to_hex(3.14))    # → c3f54840
print(float_to_hex(-1.5))    # → 0000c0bf
print(float_to_hex(0.0))     # → 00000000
```

## 命令列表

> **说明**：以下示例默认节点 ID = `0`。若节点 ID 非 0，请按 `CAN_ID = (NodeID << 7) | CMD` 计算。

---

### 一、运行时命令（0x00 ~ 0x0F）

这些命令**不会**保存到 EEPROM，掉电丢失。

---

#### 0x01 — 使能电机

让电机进入运行模式或停止。

| 字节 | 类型 | 说明 |
|------|------|------|
| 0~3 | `uint32` | `1` = 使能（速度模式），`0` = 停止 |

```bash
# 使能电机
cansend can0 001#0100000000000000

# 停止电机
cansend can0 001#0000000000000000
```

---

#### 0x02 — 执行编码器校准

触发编码器校准流程。电机将自动旋转以检测编码器安装偏移。

| 字节 | 类型 | 说明 |
|------|------|------|
| 0~7 | — | 无需数据，全填 0 即可 |

```bash
cansend can0 002#0000000000000000
```

---

#### 0x03 — 设置电流

设置目标电流，电机自动切换到**电流闭环模式**。

| 字节 | 类型 | 说明 |
|------|------|------|
| 0~3 | `float` | 目标电流（单位：**安培 A**） |

```bash
# 设置电流为 0.5A
# float 0.5 → 小端十六进制: 0000003f
cansend can0 003#0000003f00000000
```

---

#### 0x04 — 设置速度

设置目标速度，电机自动切换到**速度闭环模式**。

| 字节 | 类型 | 说明 |
|------|------|------|
| 0~3 | `float` | 目标速度（单位：**转/秒 r/s**） |

```bash
# 设置速度为 5 转/秒
# float 5.0 → 小端十六进制: 0000a040
cansend can0 004#0000a04000000000

# 设置速度为 -2 转/秒（反转）
# float -2.0 → 小端十六进制: 000000c0
cansend can0 004#000000c000000000
```

---

#### 0x05 — 设置位置（绝对位置）

设置目标位置，电机自动切换到**位置闭环模式**。位置基于零位（Home）计算。

| 字节 | 类型 | 说明 |
|------|------|------|
| 0~3 | `float` | 目标位置（单位：**转**） |
| 4 | `uint8` | `非0` = 请求应答（返回当前位置和完成状态） |

**应答帧**（当 byte4 ≠ 0 时返回）：

| 字节 | 类型 | 说明 |
|------|------|------|
| 0~3 | `float` | 当前实际位置（转） |
| 4 | `uint8` | `1` = 已到达目标，`0` = 运动中 |

应答 CAN ID = `(NodeID << 7) | 0x23`

```bash
# 转到 10 圈位置，不请求应答
# float 10.0 → 小端十六进制: 00002041
cansend can0 005#0000204100000000

# 转到 0 圈位置（回零），请求应答
cansend can0 005#0000000001000000
```

---

#### 0x06 — 设置位置（带运动时间）

设置目标位置并指定运动时长，电机在指定时间内完成运动（梯形速度曲线）。

| 字节 | 类型 | 说明 |
|------|------|------|
| 0~3 | `float` | 目标位置（单位：**转**） |
| 4~7 | `float` | 运动时间（单位：**秒**） |

> **注意**：当 byte4 的最低字节非 0 时，电机会在到达目标后返回应答帧（CAN ID = `(NodeID << 7) | 0x23`）。

```bash
# 转到 5 圈位置，用时 2 秒
# float 5.0  → 0000a040
# float 2.0  → 00000040
cansend can0 006#0000a04000000040

# 转到 -3 圈位置，用时 1.5 秒
# float -3.0 → 000040c0
# float 1.5  → 0000c03f
cansend can0 006#000040c00000c03f
```

---

#### 0x07 — 设置位置（带速度限制）

设置目标位置，并指定本次运动的最大速度限制。

| 字节 | 类型 | 说明 |
|------|------|------|
| 0~3 | `float` | 目标位置（单位：**转**） |
| 4~7 | `float` | 速度限制（单位：**转/秒 r/s**） |

> **注意**：此命令**始终**返回应答帧（CAN ID = `(NodeID << 7) | 0x23`）。

```bash
# 转到 20 圈位置，最大速度 3 转/秒
# float 20.0 → 0000a041
# float 3.0  → 00004040
cansend can0 007#0000a04100004040
```

### 三、查询命令（0x20 ~ 0x2F）

查询命令会通过 CAN 总线**返回数据**，应答帧的 CAN ID 为 `(NodeID << 7) | 命令码`。

---

#### 0x21 — 查询电流

返回当前 FOC 电流值。

**应答帧**（CAN ID = `(NodeID << 7) | 0x21`）：

| 字节 | 类型 | 说明 |
|------|------|------|
| 0~3 | `float` | 当前电流（单位：**安培 A**） |
| 4 | `uint8` | `1` = 电机已到达目标/空闲 |

```bash
# 查询电流
cansend can0 021#0000000000000000
# 应答示例: can0 021#0000803f01000000 → 电流 1.0A，已到达目标
```

---

#### 0x22 — 查询速度

返回当前实际速度。

**应答帧**（CAN ID = `(NodeID << 7) | 0x22`）：

| 字节 | 类型 | 说明 |
|------|------|------|
| 0~3 | `float` | 当前速度（单位：**转/秒 r/s**） |
| 4 | `uint8` | `1` = 电机已到达目标/空闲 |

```bash
# 查询速度
cansend can0 022#0000000000000000
```

---

#### 0x23 — 查询位置

返回当前实际位置（基于零位的绝对位置）。

**应答帧**（CAN ID = `(NodeID << 7) | 0x23`）：

| 字节 | 类型 | 说明 |
|------|------|------|
| 0~3 | `float` | 当前位置（单位：**转**） |
| 4 | `uint8` | `1` = 电机已到达目标 |

```bash

```

#### 0x24 — 查询零位偏移

返回编码器零位偏移值（原始整数值）。

**应答帧**（CAN ID = `(NodeID << 7) | 0x24`）：

| 字节 | 类型 | 说明 |
|------|------|------|


#### 0x25 — 查询温度

**应答帧**（CAN ID = `(NodeID << 7) | 0x25`）：

| 字节 | 类型 | 说明 |
|------|------|------|


### CAN 帧过滤规则

固件只处理以下 CAN 消息：
- 节点 ID 匹配本机
- 节点 ID = 0（广播）

```c
uint8_t id = (RxHeader.StdId >> 7);
uint8_t cmd = RxHeader.StdId & 0x7F;
if (id == 0 || id == boardConfig.canNodeId)
    OnCanCmd(cmd, RxData, RxHeader.DLC);
```
